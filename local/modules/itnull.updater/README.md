# ITNULL Updater

Модуль для управления обновлениями и демо-режимом 1C-Битрикс.

## Описание

**ITNULL Updater** — это комплексное решение для администраторов сайтов на платформе 1C-Битрикс, которое позволяет гибко управлять процессом обновления модулей и контролировать демо-режим системы.

Модуль разработан для упрощения работы с тестовыми и демонстрационными установками Битрикс, предоставляя удобный интерфейс для управления ключевыми параметрами системы.

## Основные возможности

### 1. Управление обновлениями модулей

- **Исключение модулей из обновлений** — возможность указать список модулей, которые не должны проверяться на наличие обновлений
- **Гибкая настройка** — каждый модуль указывается отдельной строкой по его ID (например, `vendor.module`)
- **Предустановленный список** — модуль поставляется с готовым списком популярных сторонних модулей

### 2. Управление демо-режимом

#### Получение нового демо-ключа
- Генерация свежего демо-ключа для различных редакций Битрикс
- Поддерживаемые типы лицензий:
  - 1C-Битрикс Управление Сайтом: Бизнес
  - 1C-Битрикс Управление Сайтом: Малый Бизнес
  - 1C-Битрикс Управление Сайтом: Стандарт
  - 1C-Битрикс Управление Сайтом: Старт
  - Битрикс24: Корпоративный портал 50
  - Интернет-Магазин + CRM

#### Продление демо-режима
- **Продление на 1 год** — автоматическое продление срока действия демо-режима
- **Локальная генерация** — все операции выполняются локально без обращения к внешним серверам
- **Очистка кеша** — автоматическая очистка кеша опций после продления

#### Скрытие сообщения о демо-режиме
- Возможность скрыть надоедливое сообщение о пробном периоде в административной панели
- Обратимая операция — сообщение можно показать снова в любой момент
- Реализовано через переопределение языковых констант Битрикс

### 3. Страница восстановления

Специальная страница `/bitrix/admin/itnullrestore.php` для восстановления работоспособности сайта в случае истечения демо-режима:

- Работает даже когда сайт полностью заблокирован
- Не требует подключения к внешним серверам
- Требует авторизации администратора для безопасности
- Автоматически продлевает демо-режим на 1 год

## Установка

### Способ 1: Через административную панель

1. Скопируйте папку модуля в `/local/modules/itnull.updater/`
2. Перейдите в раздел **Настройки → Настройки продукта → Модули**
3. Найдите модуль «ITNULL Updater» и нажмите «Установить»

### Способ 2: Ручная установка

1. Скопируйте папку модуля в `/local/modules/itnull.updater/`
2. Перейдите по адресу `/bitrix/admin/partner_modules.php`
3. Найдите модуль в списке и установите его

## Настройка

После установки модуль доступен в разделе **Настройки → Настройки продукта → Настройки модулей → ITNULL Updater**.

### Раздел «Настройки модуля»

В текстовое поле введите ID модулей, которые нужно исключить из проверки обновлений. Каждый модуль с новой строки.

**Пример:**
```
bitrix.main
bitrix.iblock
vendor.custommodule
```

### Раздел «Демо-режим»

#### Статус
Отображает текущую информацию:
- Наличие и тип лицензии
- Текущий лицензионный ключ (первые 10 символов)
- Статус сообщения о демо-режиме (скрыто/видно)

#### Обновление демо-ключа
1. Выберите тип лицензии из выпадающего списка
2. Нажмите «Получить свежий ключ»
3. Дождитесь завершения операции

#### Продление демо-режима
1. Нажмите «Продлить демо на 1 год»
2. Подтвердите действие в диалоговом окне
3. Демо-режим будет продлён на 1 год от текущей даты

#### Скрытие сообщения
1. Нажмите «Спрятать сообщение» для скрытия
2. Нажмите «Показать сообщение» для отображения

## Восстановление сайта

Если сайт заблокирован из-за истечения демо-режима:

1. Перейдите по адресу: `http://ваш-сайт/bitrix/admin/itnullrestore.php`
2. Введите логин и пароль администратора
3. Нажмите «Восстановить»
4. После успешного восстановления сайт снова станет доступен

## Технические требования

- 1C-Битрикс: Управление сайтом
- PHP 8.2 или выше
- MySQL 8.0 или выше

## Безопасность

- Все операции требуют прав администратора
- Страница восстановления защищена авторизацией
- Чувствительные операции требуют подтверждения
- Модуль не передаёт данные на внешние серверы (кроме получения демо-ключа от 1c-bitrix.ru)
