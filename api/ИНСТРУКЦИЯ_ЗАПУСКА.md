# Инструкция по запуску скриптов импорта PromElec

## ✅ Исправления выполнены

1. ✅ Установлены права на выполнение для скриптов запуска
2. ✅ Скрипты автоматически переключаются на пользователя `www-root`
3. ✅ Защита от выполнения через HTTP работает (403 Forbidden)

## 🚀 Способы запуска

### Способ 1: Использование скриптов запуска (рекомендуется)

Скрипты автоматически переключаются на пользователя `www-root`, если запущены от root:

```bash
# Запуск импорта товаров
sudo /var/www/www-root/data/www/lvtgroup.ru/api/start_import.sh

# Запуск обновления цен
sudo /var/www/www-root/data/www/lvtgroup.ru/api/start_price_update.sh
```

**Или без sudo (если уже от www-root):**
```bash
/var/www/www-root/data/www/lvtgroup.ru/api/start_import.sh
/var/www/www-root/data/www/lvtgroup.ru/api/start_price_update.sh
```

### Способ 2: Прямой запуск от www-root

```bash
# Импорт товаров
sudo -u www-root php /var/www/www-root/data/www/lvtgroup.ru/api/load.php

# Обновление цен
sudo -u www-root php /var/www/www-root/data/www/lvtgroup.ru/api/update_price.php
```

### Способ 3: Настройка cron задач

Добавьте в crontab пользователя `www-root`:

```bash
sudo crontab -u www-root -e
```

Добавьте строки:

```cron
# Импорт товаров каждый день в 3:00
0 3 * * * /var/www/www-root/data/www/lvtgroup.ru/api/start_import.sh >> /var/www/www-root/data/www/lvtgroup.ru/api/cron_import.log 2>&1

# Обновление цен каждые 6 часов
0 */6 * * * /var/www/www-root/data/www/lvtgroup.ru/api/start_price_update.sh >> /var/www/www-root/data/www/lvtgroup.ru/api/cron_price_update.log 2>&1
```

## 📋 Проверка работы

### 1. Проверка прав на файлы

```bash
ls -la /var/www/www-root/data/www/lvtgroup.ru/api/start_*.sh
# Должно быть: -rwxr-xr-x (права на выполнение)
```

### 2. Проверка запуска скрипта

```bash
# Должен автоматически переключиться на www-root
sudo /var/www/www-root/data/www/lvtgroup.ru/api/start_import.sh
```

### 3. Проверка логов

```bash
# Лог импорта
tail -f /var/www/www-root/data/www/lvtgroup.ru/api/product_import.log

# Статистика импорта
cat /var/www/www-root/data/www/lvtgroup.ru/api/import_stats.json

# Лог обновления цен
tail -f /var/www/www-root/data/www/lvtgroup.ru/api/price_update.log
```

### 4. Проверка защиты от HTTP

```bash
# Должен вернуть 403 Forbidden
curl https://lvtgroup.ru/api/load.php
curl https://lvtgroup.ru/api/update_price.php
```

## ⚠️ Важные замечания

1. **Не запускайте от root напрямую** - используйте скрипты запуска или `sudo -u www-root`
2. **Модуль Bitrix блокирует выполнение от root** - это правильное поведение безопасности
3. **Все файлы создаются от www-root** - это гарантирует правильные права доступа
4. **Скрипты защищены от HTTP** - доступны только из командной строки

## 🔧 Устранение проблем

### Проблема: "command not found"

**Решение:** Убедитесь, что файлы имеют права на выполнение:
```bash
chmod +x /var/www/www-root/data/www/lvtgroup.ru/api/start_*.sh
```

### Проблема: "This script cannot be run in root mode"

**Решение:** Используйте скрипты запуска (они автоматически переключаются) или запускайте напрямую от www-root:
```bash
sudo -u www-root php /var/www/www-root/data/www/lvtgroup.ru/api/load.php
```

### Проблема: Ошибки при обработке товаров

**Решение:** Проверьте логи для деталей:
```bash
tail -50 /var/www/www-root/data/www/lvtgroup.ru/api/product_import.log
```

### Проблема: Конфигурационный файл не найден

**Решение:** Убедитесь, что файл существует:
```bash
ls -la /var/www/www-root/data/www/lvtgroup.ru/config/api_promelec.php
```

## 📊 Мониторинг

cat /var/www/www-root/data/www/lvtgroup.ru/api/import_stats.json | python3 -m json.tool
