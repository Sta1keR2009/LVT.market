# Быстрое исправление критических уязвимостей

## Шаг 1: Защита основных скриптов импорта

Добавьте в начало файлов `load.php` и `update_price.php` (после установки DOCUMENT_ROOT):

```php
// Проверка на CLI-режим - разрешить только запуск из командной строки
if (php_sapi_name() !== 'cli') {
    http_response_code(403);
    die('Доступ запрещен. Скрипт доступен только из командной строки.');
}
```

## Шаг 2: Удалить или переместить тестовые скрипты

**Вариант А (рекомендуется):** Удалить файлы:
- `tovar.php`
- `vsetovary.php`  
- `debug_api.php`

**Вариант Б:** Переместить в защищенную директорию:
```bash
mkdir -p /var/www/www-root/data/www/lvtgroup.ru/api/test/
mv tovar.php vsetovary.php debug_api.php api/test/
```

## Шаг 3: Защитить скрипты изменения данных

Добавьте в начало файлов `add_price.php`, `add_stock.php`, `check_product.php`:

```php
require_once($_SERVER['DOCUMENT_ROOT'].'/bitrix/modules/main/include/prolog_before.php');

// Проверка авторизации администратора
global $USER;
if (!is_object($USER)) {
    $USER = new CUser;
}

if (!$USER->IsAuthorized() || !$USER->IsAdmin()) {
    http_response_code(403);
    die('Доступ запрещен. Требуется авторизация администратора.');
}
```

## Шаг 4: Создать .htaccess

```bash
cd /var/www/www-root/data/www/lvtgroup.ru/api/
cp .htaccess.example .htaccess
# Отредактируйте .htaccess под ваши нужды
```

Или создайте `.htaccess` вручную с содержимым:

```apache
# Разрешить доступ только с localhost
<FilesMatch "\.php$">
    Order Deny,Allow
    Deny from all
    Allow from 127.0.0.1
    Allow from ::1
</FilesMatch>

# Запретить доступ к логам и конфигам
<FilesMatch "\.(json|log|config|ini|txt)$">
    Order Deny,Allow
    Deny from all
</FilesMatch>

Options -Indexes
```

## Шаг 5: Вынести учетные данные

Создайте файл `/var/www/www-root/data/www/lvtgroup.ru/config/api_promelec.php`:

```php
<?php
return [
    'login' => 'lvtgroup2',
    'password' => '30316',
    'customer_id' => 148949,
    'api_url' => 'https://aaa.na4u.ru/rpc/',
];
```

Обновите скрипты для использования конфига (см. полный отчет).

## Проверка

После исправлений проверьте:

1. Скрипты импорта недоступны через HTTP:
   ```bash
   curl https://lvtgroup.ru/api/load.php
   # Должно вернуть 403 или "Доступ запрещен"
   ```

2. Тестовые скрипты недоступны или удалены

3. Скрипты изменения данных требуют авторизацию

4. .htaccess работает (проверьте доступ к логам)

---

**ВАЖНО:** После исправлений протестируйте запуск cron-задач:
```bash
php /var/www/www-root/data/www/lvtgroup.ru/api/load.php
php /var/www/www-root/data/www/lvtgroup.ru/api/update_price.php
```

Они должны работать из командной строки, но быть недоступны через HTTP.
